Object Oriented programming
Black Hawk

  • Role: Lead Programmer, Strategy & Ideation, UX design
  • Tools: Processing in Java , Photoshop
  • date:Summer 2018
  • Academic Project

The main objective of this project was to complete a task using given objects. The task is one where there had to be a protagonist and antagonist who had to pass different levels to defeat the boss.

Design

Each character had to be designed by ourselves along with the floor map and plan. individual width and height of the object mattered as those images would then be implemented in programming. For this I choose going with an individual commando where I created him using Photoshop along with the other characters.

Some of the images are taken from : https://unluckystudio.com/free-top-down-shooterplayer-sprites/

Programming

Due to this being OOPS concepts such as inheritance, encapsulation, polymorphism has been used. Moreover, the bounding box has been used for object detection. Following is the structure of the code.

Each enemy has a radius in which he or she can shoot if the player is present. This is made possible with the help of creating a Bounding box for object detection. Three different levels have been implemented with a boss fight in the end.

To make different levels a 3D matrix has been made where each number in the matrix corresponds to individual objects. e.g 1 is a corner block & 0 is open space where the player can move.

Challenge

There were multiple challenges while the development took place but the important one amongst them optimized the game. Let me explain! What happens is that when the game loads, in the beginning, all the images are rendered at their respective places, but reading through that 3D matrix and simultaneously loading those images when the player is moving all over the internet might not work at the same time, due to lack of resources such as computing power or the network. All this results in is Lack of User Experience.

Solution

To resolve this the camera class has been implemented in the program, what this does is that relative to the player's position it starts to load objects in the direction in which the player moves while keeping the remaining items to be loaded in the buffer memory. With the help of this technique, every object isn't loaded at once but only when it is needed on the screen.

Reflections
